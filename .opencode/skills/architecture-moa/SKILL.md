<flow>
1. task(@工具人) + task(@视觉翻译官) 搜集材料（工具人自检）
2. task(@旗舰·架构) + task(@旗舰·规划) + task(@旗舰·工程) 并行
3. task(@残差提取者) 分析差异（共识 < 95% 时）
4. task(@旗舰·融合) 三阶段融合
5. task(@旗舰·质检) 方案审查（置信度评估）
6. task(@旗舰·实现) 编码
7. task(@旗舰·质检) 代码验收 + 学习记录
</flow>
<rules>
- 系统架构或多模块设计
- 输出：质检报告
- 方案审查置信度 < 60 时打回融合层
</rules>
