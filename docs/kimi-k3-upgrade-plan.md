# Kimi K3 升级方案

> 撰于 2026-07-17 | Kimi K3 发布于 2026年7月，OpenCode Go 计划已上线

---

## 现状

| 项目 | 值 |
|------|-----|
| 当前 Kimi 模型 | `kimi-k2.7-code`，用于**中级·融合** agent |
| 定价（输出） | $4 / 1M tokens |
| 配额 | 3,200 次 / 5h |

## Kimi K3 定位

| 属性 | 值 |
|------|-----|
| 模型 ID | `kimi-k3` |
| 定价（输出） | $15 / 1M tokens（约 K2.7 的 3.75x） |
| 上下文窗口 | **1M tokens** |
| 输入方式 | 文本 + 图片 + 视频 |
| Go 计划配额 | 880 次 / 5h |
| 当前排名 | Go 计划用量 #15（标注 **New**，5B tokens） |

---

## 升级策略：三级推进

### 一级（立即发版，v0.1.0）——抢热点

直接升级 `中级·融合` 到 `kimi-k3`。

**改动清单：**

| 文件 | 改动 |
|------|------|
| `.opencode/agents/中级·融合.md` | `model: opencode-go/kimi-k2.7-code` → `model: opencode-go/kimi-k3` |
| `CHANGELOG.md` | 新增 v0.1.0 条目 |
| `README.zh.md` | 模型列表 K2.7 → K3，更新定价 |
| `README.md` | 同上 |
| `install.ps1` | 注册 `kimi-k3` 模型条目 |
| `install.sh` | 注册 `kimi-k3` 模型条目 |
| `.opencode/tests/T0-static-verify.ps1` | 更新中级·融合模型名检查 |

**理由：**
- 同是 Kimi 家族，自然延续
- 中级·融合是 MoA 瓶颈环节（三份方案→一份融合），K3 的 1M 上下文有真实价值
- 单 agent 替换，不影响其他链，风险最低
- 「支持 K3」是用户一眼能看到的卖点

**成本影响：**
- 中级链单次调用成本：K2.7 约 $0.02~0.06 → K3 约 $0.075~0.23
- 但中级链在整体 MoA 使用频次中占比不高（日常任务多走工具层/闪电侠）

### 二级（本周）——造内容

在 `docs/benchmark/` 下新增 Kimi K2.7 vs K3 对比基准：

```
docs/benchmark/
├── index.md                    # 基准测试总览
├── kimi-k2.7-vs-k3.md          # K2.7 vs K3 中级融合对比
└── results/                    # 原始结果数据（可选）
```

**测试思路：**
- 选取不同类型的融合任务：代码生成、架构设计、文案改写、数据分析
- 同一组输入→K2.7 融合 vs K3 融合→对比输出质量
- 维度：融合完整性、幻觉率、决策合理度、格式合规

**输出用途：**
- GitHub Discussion 发帖
- SkillHub 发布模型对比
- 社交媒体传播素材

### 三级（中期）——K3 旗舰链

创建可选配置变体，让高级用户发挥 K3 全部潜力：

| 方案 | 描述 | 额外成本 |
|------|------|---------|
| **A: 旗舰·融合 → K3** | 替换 qwen3.7-max（$7.50 → $15） | +$0.02~0.06/次 |
| **B: 新增「旗舰·K3 单体」** | 跳过三重并行，一个 K3 输出方案 | 单模型代替三模型，可能更省 |
| **C: 旗舰·质检 → K3** | 方案验收阶段使用 K3 的长上下文 | 按需调用 |

**实现方式：** 在 install 脚本中提供选项：
- `Standard Mode`（K2.7 中级·融合 + 现有旗舰链）
- `K3 Performance Mode`（K3 中级·融合 + K3 旗舰·融合）

---

## 风险和注意事项

1. **成本上升：** K3 输出 $15/M，预算敏感用户可能不升级。方案保留 K2.7 作为备选。
2. **配额收紧：** K3 配额 880 次/5h vs K2.7 的 3,200 次。对高频用户影响较大。
3. **成熟度：** K3 刚上线，可能有不稳定时期。建议在 `中级·融合` 中保留快速回退到 K2.7 的机制。
4. **模型名稳定性：** 确认 OpenCode Go 的最终模型 ID 为 `kimi-k3` 而非其他变体。

## 附录：为什么选择中级·融合作为第一站

- **自然继承：** 本项目已用 Kimi K2.7，升级到大版本天经地义
- **用户感知：** 在 README 和 release note 里高亮「K3 加持」，这是直接的卖点
- **风险隔离：** 只改一个 agent，不影响旗舰链和工具层。如果 K3 有问题，回退也只影响中级融合
- **最小改动：** 改一行 model 名即可发版，当天出活
